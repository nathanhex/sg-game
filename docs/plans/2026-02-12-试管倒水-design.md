# 试管倒水游戏 — 技术设计文档

**规则依据：** 本设计的游戏行为以 [试管倒水游戏-规则说明](../试管倒水游戏-规则说明.md) 为准；实现时必须满足该文档中的倾倒规则、过关条件与关卡设计约束。

**目标：** 在浏览器中实现多关卡试管倒水益智游戏，纯 HTML + CSS + JavaScript，无框架；关卡数据内置于 JS，支持选关与重新开始本关。

---

## 1. 架构概览

- **展示层：** 单一 HTML 页面，通过 DOM 展示试管、液体、按钮与关卡信息；样式集中在 CSS 中。
- **逻辑与数据层：** 单一 JS 文件，负责关卡数据、游戏状态、倾倒与过关判定、以及根据状态更新 DOM。
- **无构建步骤、无后端：** 双击打开 HTML 或通过静态文件服务器即可运行。

职责划分：HTML 只提供骨架与挂载点；CSS 负责布局与视觉；JS 持有全部状态与规则，并驱动 UI 更新。

---

## 2. 文件与模块划分

| 文件 | 职责 |
|------|------|
| `index.html` | 页面结构：试管容器、选关/重新开始/取消按钮、过关弹层等；引入 `style.css` 与 `game.js`。 |
| `style.css` | 试管与液体的视觉效果（含颜色、高度）、按钮与弹层样式、选中态与响应式布局。 |
| `game.js` | 关卡数组、当前关卡索引、当前试管状态副本；关卡校验、倾倒逻辑、过关判定；事件绑定与 DOM 更新。 |

不在第一版实现：撤销、步数统计、音效、多语言；数据与逻辑均不拆分为多文件，保持单 JS 文件便于维护。

---

## 3. 数据结构

**关卡（只读，来自关卡数组）：**

- `id`：关卡标识（可选，用于显示）。
- `capacity`：每根试管容量（正整数）。
- `tubes`：二维数组，`tubes[i]` 为第 i 根试管从**底到顶**每格的颜色 ID；空试管为 `[]`。颜色 ID 为正整数（如 1、2、3）。
- `colors`（可选）：颜色 ID 到 CSS 颜色的映射数组，如 `colors[id - 1]` 为对应十六进制颜色；缺省时可在代码中使用默认色表。

**运行时状态（可变）：**

- `currentLevelIndex`：当前关卡在关卡数组中的下标。
- `currentTubes`：当前关卡试管状态的**深拷贝**（二维数组），与 `tubes` 结构相同；所有倾倒操作只修改 `currentTubes`，不修改关卡原始数据，以便「重新开始本关」时直接重新拷贝。
- `selectedTubeIndex`：当前选中的源试管下标，无选中时为 `null`。

过关条件、倾倒规则、一次倾倒的量（顶部连续同色、目标剩余空位）均按规则文档定义实现，不在此重复。

---

## 4. 数据流与核心流程

**加载关卡：**

1. 根据 `currentLevelIndex` 取关卡对象；对关卡数据执行「关卡设计约束」校验（每种颜色总格数 ≤ capacity、至少一根试管非满或已过关、试管数 ≥ 2、数组长度与颜色 ID 有效）。校验失败则提示「关卡数据无效」并拒绝加载，不修改状态。
2. 将 `tubes` 深拷贝到 `currentTubes`；`selectedTubeIndex = null`。
3. 执行一次过关判定；若已过关，按规则文档「开局即过关」处理（如直接显示过关或提示）。
4. 根据 `currentTubes` 与 `colors` 渲染试管与液体。

**一次点击试管：**

1. 若当前无选中（`selectedTubeIndex === null`）：若该试管非空，设为选中并更新选中态 UI；若为空，按规则文档不选中或提示。
2. 若当前已选中同一根：取消选中（规则文档中的取消选择）。
3. 若当前已选中另一根：将「已选中的」视为源、「当前点击的」视为目标，执行一次倾倒尝试：
   - 若满足规则文档中的倾倒条件：计算应倾倒格数（源顶部连续同色长度与目标剩余空位的最小值），更新 `currentTubes`，清空选中，重绘；然后执行过关判定，若过关则进入过关流程。
   - 若不满足：不执行倾倒，不取消源试管的选中（按规则文档边界情况），可选提示（如「目标已满」或「只能倒入相同颜色或空试管」）。

**重新开始本关：** 使用当前关卡的 `tubes` 重新深拷贝到 `currentTubes`，清空选中，再执行一次过关判定与渲染（同上，处理开局即过关）。

**过关后：** 显示过关 UI，提供「下一关」「再玩一次」「选关」等操作；在过关状态下可忽略试管点击或保持弹层在前。

可选增强：在每次操作后检查是否存在任意合法倾倒；若不存在且未过关，提示「当前无合法移动，请重新开始本关」（规则文档中的死局处理）。

---

## 5. 错误处理与边界

- **关卡校验失败：** 不加载该关，提示「关卡数据无效」，保留当前关卡与状态不变（或回退到上一关/第一关，由产品决定）。
- **无效倾倒（目标已满、颜色不匹配、源=目标等）：** 不修改 `currentTubes`，不取消源选中，可选简短提示。
- **开局即过关：** 加载后判定一次，直接进入过关展示或视为通过。
- **死局：** 可选在每次操作后做「是否存在合法移动」检测，若无则提示并依赖「重新开始本关」。
- **选关/下一关：** 切换 `currentLevelIndex` 后按「加载关卡」流程重新校验与初始化，避免脏数据或索引越界。

实现时应对 `currentTubes` 的深拷贝、数组下标、颜色 ID 与 `colors` 下标对应关系做防御性检查，避免因数据异常导致页面报错或无法操作。

---

## 6. 测试与验证要点

- **规则一致性：** 用若干手工构造的状态（含边界）验证：倾倒是否只在规则允许时执行、倾倒量是否正确、过关判定与规则文档一致；重点验证“同色分散在两根试管时**不算过关**”。
- **关卡约束：** 故意提供违反「每种颜色 ≤ capacity」「全部试管满且未过关」的关卡数据，确认被拒绝并提示。
- **交互：** 选中后点击同一根取消、点击无效目标不取消选中、过关后点击不改变状态。
- **重新开始：** 操作若干步后重新开始，状态与首次加载该关一致；开局即过关的关卡重新开始后仍应显示过关。

## 7. 交互设计（面向 10 岁以下儿童）

**目标用户**：10 岁以下儿童。  
**设计目标**：通过**多通道、高反馈**的交互，强化「操作 → 结果」的因果关系，支持观察、试错、归纳与简单规划，发挥益智作用；反馈需**即时、明确、正向为主**，避免挫败感，激发持续尝试。

### 7.1 操作流程（儿童视角）

1. **进入关卡**：页面展示当前关卡、试管区域、大而清晰的按钮（选关、重新开始等）。可配简短引导语或图示（如「先点一根有水的试管，再点另一根，把颜色倒过去」），帮助建立心智模型。
2. **选择源试管**：点击非空试管 → **立刻**有多重反馈（见 7.2），让儿童明确「我选的是这一根」。
3. **选择目标试管**：再点击另一根 → 合法时执行倾倒并给**成功反馈**；不合法时给**温和的纠错反馈**，不批评，并保持可继续尝试。
4. **取消选择**：再次点击已选中的试管或「取消」→ 选中态消失，可选轻快提示（如「换一根试试」）。
5. **过关 / 死局**：过关时**充分庆祝**（动效+文案+可选音效）；死局时用鼓励语气引导「重新开始」，强调「再试一次就能成功」。

### 7.2 多通道反馈（充分刺激，支持益智）

每一类操作都尽量同时提供**视觉 + 动效**，可选**音效/语音**，让儿童大脑多感官参与，强化因果联系与规则学习。

| 场景 | 视觉反馈 | 动效反馈 | 可选音效/语音 | 益智作用 |
|------|----------|----------|----------------|----------|
| **选中源试管** | 试管外框加粗、高亮色（如金色/亮蓝）、轻微放大；可选顶部小图标（如「从这根倒」）。 | 选中瞬间轻微弹跳（scale）或光晕扩散；持续呼吸式高亮（缓动）。 | 短促友好音（如「叮」）；或语音「选好了，再选一根要倒进去的」。 | 建立「选中 = 当前操作对象」的对应关系。 |
| **有效倾倒** | 源试管液面下降、目标试管液面上升；可在一格倒满时加小高光。 | 液体**平滑流动感**（高度 transition，约 0.3–0.5s）；倒完后源/目标试管可轻微「落定」弹跳。 | 倒水声或轻快成功音；或「倒过去啦！」。 | 强化「选对目标 → 液体移动」的因果与规则（同色/空才能倒）。 |
| **成功过关** | 弹层 + 星星/奖杯/笑脸；过关文案大而醒目；试管区可保留并加整体高亮（如光晕）。 | 星星闪烁或依次出现、弹层缩放+淡入；可选全屏小彩带/粒子。 | 庆祝短曲或「太棒了！」「过关啦！」。 | 正向强化，促进完成欲与规划感。 |
| **无效目标（已满）** | 目标试管轻微抖动；可选红色/橙色描边 1–2 秒后消失；**不**用强烈警告色。 | 短促左右抖动（约 0.2s）；可配合「摇头」小图标。 | 轻柔提示音（非错误警报）；或「这根已经满啦，换一根试试」。 | 温和纠错，帮助归纳「满了就不能再倒」。 |
| **无效目标（颜色不匹配）** | 目标试管轻微抖动；可选黄色/橙色描边；文案或图标提示「要倒到相同颜色或空试管」。 | 同上，短促抖动。 | 「要倒到一样颜色的试管哦」或「先倒到空试管试试」。 | 帮助抽象出规则：同色或空。 |
| **点击空试管作为源** | 空试管轻微晃动或描边闪烁；不进入选中态。 | 轻微晃动。 | 「这根是空的哦，选一根有颜色的」。 | 区分「有/无」与可选对象。 |
| **取消选择** | 选中态平滑消失。 | 缩放/高亮过渡回默认态。 | 可选「换一根吧」。 | 支持试错与改选。 |
| **死局（无合法移动）** | 弹层或横幅，配鼓励插画/表情；按钮「再试一次」突出。 | 弹层淡入；避免失败感强烈的动效。 | 「没关系，再试一次吧！」「重新排一排就能成功」。 | 减少挫败，鼓励规划与重试。 |

原则：**每次操作都有可见、可听（若实现音效）的即时反馈**，让儿童能快速把「我点了什么」和「发生了什么」联系起来，促进观察、试错与规则归纳。

### 7.3 益智与认知支持

- **因果关系**：选中 → 明显高亮；倾倒 → 液体真实移动；无效 → 温和提示并保持可重试。不出现「点了没反应」或模糊反馈。
- **规则内化**：通过重复的「有效/无效」反馈与简短文案，让儿童自然归纳出「只能倒到同色或空试管」「一次倒一段同色」。
- **规划与试错**：不惩罚错误操作；无效时只给提示并保留选中态，方便换目标再试。过关与重试都给予正向或中性语气，保护信心。
- **注意力与记忆**：颜色区分明显、试管排列清晰；动效节奏适中（不过快不过慢），便于追踪「刚才倒的是哪根到哪根」。

### 7.4 可访问与输入方式（儿童适配）

- **点击区域**：试管与按钮**足够大**，建议最小约 48×48px（触屏与小手更友好）；试管之间留足间距，减少误触。
- **触屏优先**：以点击/触摸为主；避免依赖悬停（Hover）传达关键信息，因儿童常用平板/手机。
- **可选后续**：简单音效开关、语音提示开关，方便家长控制；大字体与高对比度选项，兼顾弱视或注意力需求。

### 7.5 儿童友好的文案与情绪

- **语气**：鼓励、具体、不批评。用「要倒到一样颜色的试管哦」而非「错误」；用「再试一次吧」而非「失败」。
- **按钮**：「选关」「重新开始」「下一关」「再玩一次」「取消」保持简短；可加表情或图标（如「再试一次」配笑脸）。
- **过关**：「太棒了！」「过关啦！」「你真会动脑筋！」等；可随机 2–3 句轮换，增加新鲜感。
- **纠错**：「这根已经满啦」「要倒到相同颜色或空试管哦」「这根是空的，选一根有颜色的吧」——与规则文档一致，但用对儿童友好的说法。
- **死局**：「现在动不了啦，点「再试一次」重新排一排吧」——强调「重新排」而非「你错了」。

## 8. UI 设计（面向 10 岁以下儿童）

### 8.1 设计方向与约束

- **目标用户**：10 岁以下儿童；与 §7 交互设计一致，视觉需**友好、清晰、易读**，支持益智与多反馈。
- **用途**：单页益智小游戏，桌面与移动浏览器均可玩；试管与颜色需**一眼可辨**，操作路径清晰，动效**明确但不吓人**。
- **技术约束**：纯 HTML/CSS/JS，无框架；动画以 CSS（transition/transform）为主，保证流畅与可维护。
- **儿童适配**：圆角、饱满形状、明亮但不刺眼的色彩；避免尖锐、阴暗或复杂纹理；反馈态（选中、成功、纠错）区分明显。

### 8.2 风格建议（儿童向）

- **推荐 — 柔和游戏风（儿童版）**：浅色或柔和背景；试管**略粗、圆角大**，液体**饱满、不透明、颜色鲜明**；主色+强调色明确；选中与成功用**明亮、积极**的色（如金黄、亮蓝、绿）；纠错用**温和**的橙/黄，避免刺眼红。整体偏**轻松、安全、易上手**，与 §7 的鼓励语气一致。
- **备选 — 清爽实验室风**：若希望偏「科学小实验」感，可保留浅底与细试管，但液体仍建议饱满、颜色对比强，并保证选中/反馈态足够明显，满足儿童对即时反馈的需求。

实现时选定一种并统一：主色、试管形状、液体质感、按钮与弹层风格保持一致；所有反馈态（选中、有效倾倒、过关、无效）在视觉上可区分且符合儿童审美。

### 8.3 布局与空间

- **试管区域**：居中或略偏上；试管横向排列，间距一致；单根试管宽度与容量（格数）协调，使「几格液体」一目了然。小屏时试管可适当缩小或横向滚动，保证至少 2 根试管同时可见。
- **操作区**：选关、重新开始、取消等置于顶部横条或底部固定条，与试管区域有明确留白，不重叠。
- **过关/死局**：居中弹层 + 半透明遮罩；弹层内标题 + 说明 + 按钮纵向排列，主按钮（下一关/再玩一次）更突出。

### 8.4 组件与视觉规范

- **试管**：竖长条，底部可略圆或平；内部按格数划分，每格一种颜色；空格可为透明或与「空」态背景区分。试管外框与液体边界清晰，便于区分不同颜色。
- **液体**：同一试管内同色连续格可画成一段连续色块（中间无缝隙）；颜色来自关卡 `colors` 或默认色表，保证对比度与色盲可辨（可选后续优化）。
- **选中态**：与外框/阴影/缩放之一结合，与默认态对比明显；不依赖单一颜色变化，兼顾色盲。
- **按钮**：与整体风格一致；主操作（下一关、再玩一次）更醒目，次要（取消）弱化。
- **选关**：列表或网格展示关卡序号/缩略图；当前关高亮；点击即切换并加载。

### 8.5 动效与细节（儿童友好）

- **倾倒**：源试管液体减少、目标试管液体增加，用 CSS transition 做高度变化；时长约 **0.3–0.5s**，让儿童能清楚看到「液体从哪到哪」，强化因果与规则理解；节奏不宜过快。
- **选中/取消**：试管选中态用 transition 过渡，约 0.15–0.2s；可选轻微弹跳或光晕，增强「我选了这个」的确认感。
- **过关弹层**：星星/奖杯/笑脸等元素可**依次出现**（stagger）或轻微闪烁，延长一点庆祝时间，增强成就感；避免一闪而过。
- **纠错动效**：无效目标时的抖动等保持**短而温和**，不吓人、不刺眼。
- **字体**：圆角无衬线体更友好；字号足够大，按钮与关卡序号在移动端也易读；可选略带趣味感的字体与游戏调性一致。

### 8.6 响应式与极端情况

- 窄屏：试管可缩小或减少同时展示根数，保证可点、可辨；按钮可纵向堆叠或缩小字号。
- 试管很多（如 8 根以上）：考虑横向滚动或换行，避免单行过长；选关列表同理，可分页或滚动。

交互与 UI 实现时以规则文档和技术设计中的数据流为准；视觉与文案若有调整，可在本小节内增补，保持设计文档为唯一入口。
