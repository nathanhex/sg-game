# 试管倒水游戏 — 100 关规划设计

**日期：** 2026-02-14  
**依据：** [试管倒水游戏-规则说明](../试管倒水游戏-规则说明.md)、[技术设计](2026-02-12-试管倒水-design.md)

---

## 1. 目标与约束

**目标：** 为游戏规划并实现 100 个关卡，形成清晰难度递进，保证每关可解且符合现有校验。

**约束：**

- 每关通过 `validateLevel` 校验：试管数 ≥ 2、每种颜色总格数 ≤ capacity、至少一根非满或已过关、单管长度 ≤ capacity。
- 每关必须**可解**（存在从初始状态到过关状态的合法操作序列）。
- 数据结构与现有一致：`{ id, capacity, tubes, colors }`，`tubes` 为底到顶颜色 ID 的二维数组，`colors` 使用项目 `PALETTE` 前 K 色。

**不在此版实现：** 步数限制、星级、章节分组 UI；仅扩展 `LEVELS` 数组至 100 条。

---

## 2. 难度维度与分段

**难度维度：**

- **颜色数**：越多则目标状态越复杂，可选移动越多，规划难度上升。
- **试管数**：空管越多缓冲越大，通常越容易；试管多且空管少则更紧。
- **初始搅乱程度**：同色分散越厉害、顶层颜色越杂，需要更多步骤与更优顺序才能解开。

**分段与参数（capacity 统一为 4）：**

| 段落       | 关卡号   | 颜色数 | 试管数 | 空管数 | 说明           |
|------------|----------|--------|--------|--------|----------------|
| 入门       | 1–20     | 2      | 3–4    | 1–2    | 2 色为主，少量 3 色 |
| 进阶       | 21–50    | 2–3    | 4–5    | 1–2    | 引入 3 色，试管略增 |
| 困难       | 51–80    | 5–6    | 6–7    | 1      | 5–6 色、仅 1 空管、搅乱步数 32–43 |
| 挑战       | 81–100   | 6–7    | 7–8    | 1      | 6–7 色、仅 1 空管、搅乱步数 44–57 |

现有第 1–3 关保留为前 3 关模板（2 色 / 2 色 / 3 色），4–100 关按上表由程序生成，保证可解与校验通过。

---

## 3. 关卡数据格式与可解性保证

**数据格式：** 与现有一致。

```js
{
  id: 1,
  capacity: 4,
  tubes: [[1,2,1],[2,1,2],[]],  // 底→顶，空管 []
  colors: ["#FF6B6B","#4ECDC4"]
}
```

**可解性保证策略：** 采用「候选局面生成 + 求解器验收」生成初始状态。

1. **候选局面**：先直接构造满足颜色计数、容量、空管数量与档位混合度要求的开局，而不是假设某个“反向搅乱”过程天然成立。
2. **混合度控制**：在候选生成阶段限制高难关卡的连续同色长度、整管同色数量与模板化结构，避免大量 `[a,b,b,b]` 形式的重复布局。
3. **求解验证**：使用与 game.js 一致的正向倒水规则运行求解器；只有求解成功的候选局面才接受，因此可解性来自验证而非伪反向构造。
4. **尽量少出现「开局整管同色」**：在候选评分时优先保留整管同色数量更少、已归位管更少的局面，使开局更混合、避免大量试管一开始就已归位。
5. **禁止「进入即过关」**：生成结束后用与 game.js 一致的 `isLevelComplete` 判定；若为过关状态则直接丢弃并重生，避免选关进入后直接弹出过关。

**颜色与试管数：** 按上表为每关选定 `numColors`、`numTubes`；空管数 = `numTubes - numColors`（每种颜色 4 格，共 `numColors` 根满管，其余为空）。

---

## 4. 实现方式

- **关卡 1–3**：保留现有手写数据，仅保证 `id` 与顺序一致。
- **关卡 4–100**：由独立脚本或 `game.js` 内联逻辑，按段落表为每关设定 `(numColors, numTubes)`，直接构造候选混合局面并用正向求解器验收，得到 `tubes`；`colors = PALETTE.slice(0, numColors)`。
- 生成后对每关调用 `validateLevel(level)`，不通过则重新生成或调整参数。
- 最终将 100 个关卡对象写入 `game.js` 的 `LEVELS` 数组，替换现有 3 关数组。

**验收：** 加载 1–100 关无校验报错；选关、重新开始、过关后下一关行为正常；前几关明显更简单、后段关卡需要更多步骤与规划。
