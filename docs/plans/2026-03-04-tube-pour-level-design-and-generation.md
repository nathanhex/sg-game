# 倒水游戏 — 关卡设计与生成规则

**日期：** 2026-03-04  
**目标：** 在保留可解性的前提下，解决高难度关卡中「大量连续 3 格同色」导致的难度偏低问题，并统一关卡设计约束与生成规则。  
**读者：** 实现/改造生成工具与诊断工具时查阅。

---

## 1. 目标与约束

### 1.1 目标

- **难度与档位匹配**：困难/非常困难关卡应显著减少「连续 3 格同色」，优先出现「最多连续 2 格」；简单/中等可放宽。
- **可解性**：关卡以「候选局面生成 + 正向求解器验收」保证可解，不再把“过关种子 + 伪反向搅乱”作为硬前提。
- **可验证**：生成规则与质量指标明确，诊断工具与求解器都可复现指标。

### 1.2 约束

- 沿用现有数据结构与 `validateLevel`、`isLevelComplete`（见 `docs/LEVEL-DATA-SPEC.md`）。
- 不改变游戏规则（capacity=4、倒水规则不变）。
- 若未来继续尝试“反向构造”，必须实现并验证**真正的逆操作**；禁止复用正向倒水合法性来伪装成反向步骤。

## 2. 问题与对策总览

### 2.1 问题清单（含解决方案与规则规避）

下表汇总所有已发现问题：**解决方案**说明怎么做，**规则/算法**说明是否以及如何通过生成规则从源头规避。

| 编号 | 问题描述 | 解决方案 | 能否用规则规避 | 对应规则/算法 |
|------|----------|----------|----------------|----------------|
| **P1** | 开局就有整管同色 | 候选评分时优先保留「整管同色数更少」的局面；困难/非常困难目标为 0 | 能 | 候选评分最小化整管同色数（§4.2、§4.5） |
| **P2** | 开局即通关 | 搅乱后若 `isLevelComplete` 为 true 则重试（换 seedOffset 或减步数） | 能 | 生成后必检 + 重试（§5.4） |
| **P3** | 困难段已归位过多、难度偏低 | 通过 P1+P4 联合改善；诊断保留「已归位管」指标 | 间接能 | 由 P1+P4 步选与门控自然满足，无需单独规则 |
| **P4** | 高难度关卡大量连续 3 格同色 | 引入 max 连续段 ≤ 2；候选构造优先、门控重试 | 能 | 候选生成约束 `maxSegmentLength≤2` + 门控（§4.2、§4.5） |
| **P5** | 生成只约束整管、未约束连续段 | 实现 maxSegmentLength，候选生成与门控都用 | 能 | 与 P4 同一套实现（§4.2、§5.1） |

| **P7** | 仅靠非过关混合种子虽然能生成局面，但缺少可解性保证 | 保留“直接生成混合局面”，但必须追加求解器验收 | 能 | 硬性规则：所有候选局面必须求解通过（§4.3） |
| **A1** | 解长过短（体感偏易） | 搅乱步数下限；可选：求解器+最小解长门控 | 部分能 | 步数下限已有；可选求解器门控 |
| **A2** | 候选生成容易集中到少数固定模板 | 在构造阶段加入多样化约束，并按评分选择候选 | 能 | 候选评分 + 去模板化约束（§4.2、§4.4） |
| **A3** | 门控重试失败、死循环 | 重试上限 + 接受当前最优或标记未达标 | 部分能 | 求解与评分分离；上限兜底（§4.4） |
| **A4** | 难度曲线断层 | getScrambleSteps 单调平滑；生成后抽查、单关重生成 | 部分能 | 步数公式平滑；可选后处理 |
| **A5** | 可解性需程序验证 | 可选：诊断/CI 中跑求解器，无解则报错 | 能 | 构造即保证；求解器用于发现 bug |
| **A6** | 空管过多导致过易 | 困难/非常困难空管数 ≤ 1 | 能 | 档位参数 numTubes = numColors+1（§3.3） |
| **A7** | 生成器版本与复现混淆 | 文档/注释标明 LEVELS 来源与生成时间 | 不适用 | 流程与文档 |

### 2.2 诊断报告与问题对应

| 诊断输出项 | 对应问题 | 验收方式 |
|------------|----------|----------|
| 开局整管同色 | P1 | 困难/非常困难：整管同色数=0；简单/中等：尽量少 |
| 开局即通关 | P2 | 所有关卡 `isLevelComplete(level.tubes)` 均为 false |

- **推荐**：A5（可解性求解器验证）、A7（版本注明）。
- **可选**：A1（解长门控）、A4（难度平滑）。

## 3. 混合度与档位参数

### 3.1 混合度指标定义

- **单管最大连续同色长度（max segment length）**：一根试管内同一颜色连续出现的最大长度。例：`[1,1,2,1]`→2；`[2,2,2,1]`→3。
- **顶部段长度**：试管顶部同色连续格数（一次可倒出的量）。例：`[1,2,2,2]`→3。
- **整管同色数**：满足「长度≥2 且整管同色」的试管数量（与 `countSingleColorTubes` 一致）。

### 3.2 按档位的混合度要求

| 档位     | 关卡 ID  | 单管最大连续同色 | 整管同色数 |
|----------|----------|------------------|------------|
| 简单     | 1–25     | ≤4（不额外限制） | 尽量少     |
| 中等     | 26–55    | ≤3               | 尽量少     |
| 困难     | 56–80    | ≤2               | 0          |
| 非常困难 | 81–100   | ≤2               | 0          |

### 3.3 档位与参数（颜色数、试管数、搅乱步数）

| 档位     | 关卡 ID | 颜色数 | 试管数 | 搅乱步数（参考） |
|----------|---------|--------|--------|------------------|
| 简单     | 1–25    | 2      | 3–4    | 4–8              |
| 中等     | 26–55   | 2–3    | 4–5    | 8–16             |
| 困难     | 56–80   | 3–5    | 4–6    | 16–35            |
| 非常困难 | 81–100  | 5–7    | 6–8    | 35–55            |

困难/非常困难空管数 = numTubes − numColors = 1。随机/自定义模式使用同一套档位与生成规则。

## 4. 生成规则（统一）

### 4.1 候选局面

- 生成器的直接产物是**候选开局**，而不是“必须来自某个过关种子的搅乱结果”。
- 候选开局必须满足：
  - 颜色总数正确：每种颜色出现次数 ≤ `capacity`，且目标关卡使用的每种颜色都按预期出现；
  - 结构合法：每根试管长度 ≤ `capacity`，且至少一根试管未满；
  - 非开局即通关：`isLevelComplete(level.tubes)` 必须为 false。
- 候选局面可以通过以下方式产生：
  - 直接按颜色池分配到各试管；
  - 在某个基础混合布局上做局部交换、重排或扰动；
  - 其他离线构造方法。

### 4.2 候选生成策略

候选生成不再依赖伪“反向倒水”，而是在构造阶段直接控制混合度：

1. **档位参数**：先按关卡 ID 确定颜色数、试管数与目标难度。
2. **颜色分散**：同一种颜色应尽量分散到多根试管，避免天然形成 `3` 连或 `4` 连。
3. **连续段约束**：
   - 简单/中等：允许较松的连续段上限（如 ≤3）。
   - 困难/非常困难：将 `maxSegmentLength ≤ 2` 作为硬门槛。
4. **整管同色约束**：全档位都应尽量减少单色管；困难/非常困难目标为 0。
5. **去模板化**：候选评分时要惩罚大量重复的固定形态，例如多根管同时出现 `[a,b,b,b]`、`[b,c,c,c]` 之类的链式模板。

### 4.3 可解性验收

每个候选局面都必须经过**正向求解器**验收：

1. 使用与游戏内完全一致的正向倒水规则枚举合法移动。
2. 求解器需从候选开局搜索到一个过关状态；找不到解的候选局面一律丢弃。
3. 若候选局面 `validateLevel` 不通过、`isLevelComplete` 为 true、或求解器超出安全上限仍未找到解，也一律丢弃。
4. 可选：记录最短解长或近似解长，用于进一步筛掉“虽然可解但太短、体感过易”的候选局面。

### 4.4 禁止的伪反向构造

- 如果未来要重新引入“过关种子 + 反向构造”，必须满足：
  - 逆操作与正向规则严格互逆；
  - 从过关状态出发能真实地产生更混合的中间局面；
  - 有独立测试证明该逆操作不会退化成“整管单色在空管之间搬运”。
- 在此之前，**禁止**把“正向合法倒水条件”直接复用到 `getReversePourOptions` 一类函数中充当反向步骤。

### 4.5 质量门控与兜底

- **门控**：困难/非常困难（id≥56）候选生成后，若任意试管存在长度≥3 的连续同色，则丢弃本关并重试。
- **求解门控**：求解器未找到解、或达到搜索上限仍无结论，则丢弃并重试。
- **兜底**：达到重试上限（如 300）后，接受“当前评分最优且已求解通过”的那一版；若没有任何求解通过的候选，则标记失败并停止生成，避免悄悄退回低质量方案。

## 5. 实现要点

### 5.1 生成工具必须实现

- **maxSegmentLength(tubes)**：返回所有试管中最大连续同色长度。
- **候选构造器**：负责按档位参数直接构造混合开局，而不是依赖伪反向搅乱。
- **候选评分**：至少考虑 max segment、整管同色数、已归位管数、模板重复度。
- **正向求解器**：按真实游戏规则验证候选局面的可解性。
- **门控与兜底**：按 §4.5；只有“求解通过”的候选才可进入最终比较。

### 5.2 诊断工具建议

- 每关输出：max 连续段长度、整管同色数、是否满足当前档位混合度（如 id≥56 时 max≤2）。
- 增加可解性检查（求解器），无解则报错。
- 可选：输出最短解长或近似解长，用于评估“可解但偏易”的问题。
- 便于批量检查现有 100 关及重新生成后的对比。

---

## 6. 验收标准

- 困难（56–80）、非常困难（81–100）：任意试管不存在长度≥3 的连续同色；整管同色数为 0。
- 所有关卡都必须 `validateLevel` 通过、不是开局即通关，且经正向求解器验证为可解。
- 诊断工具能输出每关的 max segment length、整管同色数、档位符合性，以及可选的求解结果。
- 若继续保留「改进版」脚本，其输出也必须经过同一套求解器验收；未经过求解验证的“分散布局 + 洗牌”脚本不能作为权威生成器。

## 附录 A. 其他考虑（简要）

以下为补充说明，细节已融入 §2 问题表或 §4 生成规则，此处仅作索引。

- **解长与真实难度（A1）**：混合度为难度代理；可选求解器+最小解长门控。
- **搅乱提前终止（A2）**：步选优先保留后续选项；实际步数过小则重试。
- **门控重试失败（A3）**：重试上限 + 接受最优或标记未达标。
- **难度曲线（A4）**：getScrambleSteps 单调平滑；生成后抽查、必要时单关重生成。
- **可解性验证（A5）**：构造即保证；可选求解器发现实现 bug。
- **空管过多（A6）**：档位参数已约定困难/非常困难仅 1 空管。
- **版本与复现（A7）**：文档/注释标明 LEVELS 生成器版本与生成时间；前几关若手写可在脚本中跳过或固定。
