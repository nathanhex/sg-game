# 试管倒水游戏 — 游戏规则说明

## 1. 游戏概述

玩家面对若干根**试管**，每根试管中装有若干格**有色液体**。通过在不同试管之间**倾倒液体**，使相同颜色的液体归到同一根试管中。当每根试管内**最多只有一种颜色**（或为空），且**同一种颜色的液体只出现在同一根试管中**时，本关**过关**。

---

## 2. 游戏元素

### 2.1 试管（Tube）

- 每根试管有固定的**容量**（例如 4 格），即最多能装几格液体。
- 液体按**格**从下往上堆放；同一格内只有一种颜色。
- 试管可以**空**，也可以**部分装满**或**装满**。
- 同一关中，所有试管的容量相同。

### 2.2 液体 / 颜色

- 每格液体具有一种**颜色**，用颜色 ID（1、2、3…）表示，对应一种显示颜色（如红、蓝、绿）。
- **相同颜色**的液体可以混合：从一根试管倒入另一根时，只有与目标试管**顶部颜色相同**（或目标为空）时才能倒入，且倒入后视为同一颜色，可继续接收同色液体。
- 不同颜色的液体不能在同一格混合；倾倒时必须遵守「目标试管顶部与要倒的液体同色或目标为空」的规则。

### 2.3 关卡

- 一关由多根试管组成，每根试管的初始状态（从底到顶每格的颜色）在关卡数据中给定。
- 通常会有 1～2 根**空试管**作为缓冲，便于玩家移动液体。
- 颜色种类数、试管数量、每根试管容量由关卡设计决定。

---

## 3. 操作方式

### 3.1 基本操作：倾倒（Pour）

1. **选择源试管**：玩家点击一根**非空**试管，表示要从中倒出液体。
2. **选择目标试管**：玩家再点击另一根试管，表示要把液体倒进该试管。

完成一次「源 → 目标」的选择后，若满足下述**倾倒规则**，则执行倾倒并更新画面；若不满足，可忽略本次操作或给出简单提示（如目标无效），不改变状态。

### 3.2 取消选择

- 若已选中源试管，再次点击**同一根试管**或点击「取消」按钮，则取消当前选择，不执行倾倒。
- 若已选中源试管后点击**另一根试管**，则按「源 → 目标」尝试执行倾倒（见下）。

---

## 4. 倾倒规则（何时允许倒）

一次倾倒**允许执行**，当且仅当以下条件**全部**满足：

1. **源试管非空**  
   源试管至少有一格液体。

2. **目标试管与源试管不是同一根**  
   不能把液体「倒回自己」。

3. **目标试管未满**  
   目标试管当前格数小于容量，至少还能放 1 格。

4. **颜色匹配**  
   - 要么**目标试管为空**，此时可以倒入源试管顶部的**任意一种**颜色；  
   - 要么**目标试管非空**，且目标试管**顶部**的颜色与源试管**顶部**的颜色**相同**，才能把源试管顶部的液体倒入。

5. **一次倾倒的量**  
   - 从源试管**顶部**开始，只能倒**连续的同色液体**（一整段）。  
   - 例如源试管从顶到底为「红、红、蓝」，则一次最多倒 2 格红色到目标，不能只倒 1 格或把红色和蓝色一起倒。  
   - 倒入目标试管的格数 = min(源试管顶部同色连续格数, 目标试管剩余空位数)。  
   - 即：能倒多少就倒多少，直到这段同色液体倒完，或目标试管被装满。

**总结**：每次操作 = 把源试管**顶部一段同色液体**整段倒入**目标试管**，且目标要么为空要么顶部同色，且目标有空位。

---

## 5. 过关条件（胜利条件）

当**所有**试管都满足以下之一时，本关**过关**：

- 试管为**空**；或  
- 试管内**只有一种颜色**的液体（可以有多格，但全是同一颜色）。

并且同时满足以下跨试管约束：  

- **同一种颜色的液体必须全部集中在同一根试管中**（不能分散在两根及以上试管）。  
- **同一关中，一种颜色最多只能填满一只试管**（等价于该颜色总格数不超过单管容量）。  

即：既不允许单根试管混色，也不允许同色分散在多根试管。  
过关后，可显示过关提示，并提供「下一关」「再玩一次」「选关」等选项。

---

## 6. 边界与特殊情况

| 情况 | 处理方式 |
|------|----------|
| 点击空试管作为源 | 无效，不选中或提示「该试管为空」。 |
| 目标试管已满 | 不执行倾倒，可提示「目标试管已满」。 |
| 目标试管顶部颜色与源顶部不同且目标非空 | 不执行倾倒，可提示「只能倒入相同颜色或空试管」。 |
| 源与目标相同 | 不执行倾倒，可视为取消选择。 |
| 只倒了一部分（目标满了，源顶还有同色） | 允许；本次倾倒结束，下次可继续把剩余同色倒入其它合规目标。 |
| 过关后继续点击 | 可忽略操作，或始终显示过关界面直到玩家选择「下一关」等。 |
| 第二次点击的目标导致无效倾倒（如目标已满、颜色不匹配） | 不执行倾倒，**不取消**源试管的选中状态；玩家可继续选其他目标，或点击源试管/取消按钮取消选择。 |
| 开局即已满足过关条件 | 加载关卡后先做一次过关判定；若已过关，直接显示过关或视为本关通过（可自动进入下一关或提示「本关已完成」）。 |
| 玩家操作后陷入无合法移动且未过关 | 视为僵局（死局）。建议提供「重新开始本关」让玩家从初始状态再试；可选：检测到无任何合法移动时提示「当前无合法移动，请重新开始本关」。 |

## 7. 关卡设计约束（防止不可过关）

以下约束用于**关卡设计/数据校验**，保证关卡在理论上可解，避免用户永远无法过关：

1. **每种颜色的总格数 ≤ 试管容量**  
   否则该颜色无法全部装入一根试管，无法达成「同色必须集中在同一根试管」的胜利条件。  
   - 校验：对每种颜色 ID，统计所有试管中该颜色的格数，必须 ≤ capacity。

2. **至少有一根试管非满（或开局即过关）**  
   若所有试管都满，则没有任何空位可接收液体，无法进行任何倾倒（除非开局已过关）。  
   - 因此：要么至少一根试管有空位（包括空试管），要么初始状态已满足过关条件。

3. **建议至少一根空试管**  
   经典设计中会保留至少一根完全空的试管作为缓冲，否则很多布局会难以甚至无法解开。做关卡时建议至少一根 `tubes[i] = []`。

4. **试管数量 ≥ 2**  
   只有一根试管时无法执行倾倒（源与目标不能相同），因此单试管关卡只能是「已过关」的展示关，否则无解。

5. **关卡数据一致性**  
   - 每根试管的数组长度 ≤ capacity；  
   - 出现的颜色 ID 均在约定的颜色集合内（如 1 到 K）；  
   - 若提供 `colors` 数组，则颜色 ID 应对应有效下标（如 id 对应 `colors[id-1]`）。

实现时可在加载关卡时做上述校验，不通过则拒绝加载并提示「关卡数据无效」，避免不可过关或运行时错误。

---

## 8. 关卡流程与 UI 行为（建议）

- **开始本关**：根据关卡数据初始化每根试管的液体，显示当前关卡序号/名称；**先做一次过关判定**，若开局已过关则按「边界与特殊情况」处理。
- **进行中**：玩家通过「选源试管 → 选目标试管」进行倾倒；可提供「重新开始本关」重置到本关初始状态。
- **过关**：弹出或展示过关提示，提供「下一关」「再玩一次」「选关」。
- **选关**：可从关卡列表选择任意已开放关卡（第一版可全部开放）。

---

## 9. 规则小结（给实现用）

- **状态**：用二维数组表示每根试管从底到顶每格的颜色 ID；空格可用 0 或空位表示。  
- **一次操作**：选源试管 → 选目标试管；若满足上述倾倒规则，则把源顶部连续同色整段倒入目标（最多填满目标空位）。  
- **胜负**：每根试管要么空要么只有一种颜色，且同色只能出现在一根试管中 → 过关。  
- **取消**：选中源后点击同一根或取消按钮 → 清空选择，不倾倒。  
- **关卡加载**：建议按「关卡设计约束」做校验，不通过则拒绝加载并提示，避免不可过关或数据错误。
